# 知识图谱问答模块 (QA)

## 1. 模块概述

知识图谱问答模块（QA）是一个基于 Neo4j 知识图谱的智能问答系统，采用Think on Graph推理方法，结合大语言模型实现多跳推理和知识检索。该模块支持自然语言问题理解、实体链接、知识图谱检索和答案生成。

### 核心特性

- **多跳推理**：支持基于知识图谱的深度推理，自动探索实体间的关系路径
- **实体链接**：自动识别问题中的实体并链接到知识图谱
- **大模型驱动**：利用 LLM 进行关系选择、实体评分和答案生成
- **可视化支持**：生成推理路径的可视化图谱
- **推理日志**：记录完整的推理过程，支持可追溯性分析

## 2. 算法详情

### 2.1 核心算法：Think on Graph 推理

**算法流程**：

## 主流程
1. 问题输入 ：接收用户提出的自然语言问题
2. 问题定义 ：将自然语言问题转化为结构化的问题模型
3. 问题分解 ：将复杂问题分解为多个子问题
4. 方案规划 ：为每个子问题生成解决方案
5. 答案生成 ：基于解决方案生成最终答案
## 分支流程
### 问题定义阶段
- 实体识别与链接 ：识别问题中的实体并链接到知识图谱中的对应实体
### 问题分解阶段
- 子问题生成 ：将原始问题分解为多个可管理的子问题
### 方案规划阶段
- 知识图谱检索（多跳推理） ：
  - 关系搜索与剪枝 ：搜索与问题相关的关系，并剪枝掉不相关的关系
  - 实体搜索与评分 ：基于剪枝后的关系搜索相关实体，并对实体进行评分


### 2.2 关键组件


| **ProblemSolver** | `problem_solver.py` | 问题求解器，协调整个问答流程 |
| **EntityLinker** | `entity_linker.py` | 实体链接器，将文本实体映射到图谱实体 |
| **Neo4jClient** | `neo4j_client.py` | Neo4j 图数据库客户端，支持多服务器查询 |
| **WikiFunc** | `wiki_func.py` | ToG 推理核心函数，包括关系搜索、实体搜索、剪枝等 |
| **Neo4jQAService** | `answer_solve.py` | 问答服务封装类，提供高级接口 |
| **Visualizer** | `neo4j_visualizer.py` | 知识图谱可视化工具 |


## 3. 接口说明

### 3.1 Python 接口

#### Neo4jQAService 类

**位置**：`QA/answer_solve.py`

```python
from QA.answer_solve import Neo4jQAService
from QA.config import config

# 初始化服务
service = Neo4jQAService(config)

# 基础问答
result = service.answer_question("列车A发生故障的原因是什么？")

# 带可视化的问答
result = service.answer_question_with_visualization(
    "列车A发生故障的原因是什么？",
    visualize=True,
    visualization_type="pyvis"  
)
```

**方法说明**：

| `answer_question` | `question: str` | `Dict` | 基础问答接口，返回答案和求解结果 |
| `answer_question_with_visualization` | `question: str, visualize: bool, visualization_type: str` | `Dict` | 带可视化的问答，返回包含可视化HTML路径 |

#### ProblemSolver 类

**位置**：`QA/problem_solver.py`

```python
from QA.problem_solver import ProblemSolver
from QA.neo4j_client import MultiServerWikidataQueryClient
from QA.config import config

# 初始化客户端
client = MultiServerWikidataQueryClient(
    neo4j_uri="bolt://localhost:7687",
    username="neo4j",
    password="password"
)

# 初始化求解器
solver = ProblemSolver(client, config)

# 执行问题求解
result = solver.solve("列车A发生故障的原因是什么？")
```

**方法说明**：

| `solve` | `user_input: str` | `Dict` | 主求解流程，返回完整的求解结果 |
