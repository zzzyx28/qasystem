2.1.2.技术路线

多源数据预处理处理流程可清晰地划分为三个阶段。

（1）解析与提取
用户在前端上传文件，后端完成文件类型校验（当前支持 .pdf/.docx/.doc/.xlsx/.xls/.csv）与临时落盘。随后，统一调度器根据文件扩展名将任务分发至对应的解析器。为保障接口响应稳定，并发执行通过 asyncio.to_thread 进行线程化封装。各解析器分工明确：PDF 文件由 MinerU 处理，输出结构化 Markdown；Word 文件（.docx 直接，.doc 先经 LibreOffice 转换）由 Docling 处理，输出 Markdown；Excel/CSV 文件由 Pandas 数据引擎解析，提取结构化表格数据。

为提升可解释性，各处理模型原理如下：
- Docling：采用“文档结构感知 + 版面语义重建”思路，对 Word 文档中的段落、标题层级、列表、表格与图片引用关系进行统一解析，再按语义块重组为标准 Markdown。对于 .doc 文件，先经 LibreOffice 转换为 .docx，以降低旧格式二进制差异对结构抽取的影响。
- MinerU：面向 PDF 的多阶段解析引擎，先进行页面级版面分析（如文本区、表格区、公式区等区域定位），再执行 OCR/文本抽取与元素识别，最后按阅读顺序进行内容重排与结构化输出。该机制对扫描件、复杂排版和混合图文文档具有更强鲁棒性。
- Pandas：基于 DataFrame 的表格化数据解析机制，通过编码识别、分隔符推断与列类型推断完成 Excel/CSV 的结构恢复；随后以统一二维表结构输出，便于后续标准化转换为 Markdown 表格与下游数据库入库。

（2）标准化与输出
各解析器提取的原始材料被转换为统一的结构化输出格式，但处理逻辑各不相同。不同解析器的输出在服务层进行统一收敛。PDF 文件解析结果由 pdfprocesser 处理器生成结构化 Markdown；Word 文件解析结果由 docxprocesser 处理器生成 Markdown；Excel 文件解析则由 excelprocesser 处理器先提取结构化表格数据，再转换为标准 Markdown 表格（含表头、数据行，并对超长内容进行截断提示）。对于大文本结果，系统进行长度裁剪并附加 content_preview 字段，有效降低前端渲染与传输压力。通过上述处理，确保来自不同来源的文档输出具有一致的结构化程度和元数据完整性，为下游应用提供可靠接口。

（3）存储与应用
最终产出的统一结构化数据，具备极强的平台适配性，可被无缝注入各类存储系统，从而激活多样化的高级应用场景。数据切片后可存入知识图谱，以构建实体、属性及其间的复杂关系网络，支撑深度推理与关联发现；也可导入向量数据库，通过将文本转化为嵌入向量，赋能基于语义相似度的智能检索、问答和推荐系统；同时，其中规整的表格数据可直接落地到关系型数据库，满足传统的统计分析、报表生成和业务系统集成需求。这一设计使得原始文档中的信息得以转化为可灵活调用、可深度挖掘的数字资产，全面服务于知识管理、智能分析与决策支持。
