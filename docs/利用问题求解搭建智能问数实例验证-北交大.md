## 利用问题求解搭建智能问数实例（以一个问题作为示例，其他问题类似，保证其他相关问题也能按照问题求解流程去问）
以采购管理域智能问数搭建为实例，利用问题求解进行实例化。
### 问题定义
- 输入问题：公开采购与非公开采购金额构成如何？
- 按照所设计的问题本体模型，输出结构化问题模型。
### 问题分解
- 对于复杂问题，利用问题分解算法形成分解后的问题模型，分解后的每个节点，与定义的问题本体保持一致。
- 形成的分解结果如下：子问题1-怎么识别这个问题的意图；子问题2-怎么根据意图找上下文并进行指标匹配；子问题3-怎么转换成SQL；子问题4-怎么执行SQL；子问题5-怎么进行图表可视化和答案描述。
### 方案规划
- 从知识地图（知识图谱）中搜索到的方案路径为：
	- 意图识别：输入自然语言问题，输出结构化意图；
	- 上下文获取与指标匹配：输入结构化意图，输出用指标描述的业务；
	- SQL转换：输入指标描述的业务，输出SQL描述的业务；
	- SQL执行：输入SQL，输出指标计算结果；
	- 图表可视化与答案生成：输入计算结果，输出图表+文字内容描述。
### 物理组件匹配
- 组件1：意图识别组件
- 组件2：上下文获取与指标匹配组件
- 组件3：SQL转换组件
- 组件4：SQL执行
- 组件5：图表与答案生成组件
### 智能体执行过程
- **意图识别执行结果，输出结构化意图：**
	- Domain：采购管理域
	- Scenario：采购结构分析
	- Object：采购金额结构
	- Goal：分析公开与非公开采购金额构成
	- IntentType：看构成
- **上下文获取与指标匹配执行结果：**
	- 根据业务模型识别到该问题对应的业务活动是：采购执行结果统计、采购方式结构分析；
	- 根据指标模型匹配：`CG_001 电子采购中标金额`、`CG_003 公开采购中标金额`
	- 推导：非公开采购金额 = `CG_001 - CG_003`
	- 因此，业务被转化为用指标描述的业务：对电子采购总金额和公开采购金额进行计算，并进一步推导非公开采购金额，分析两者的金额构成与占比。
- **SQL转换执行结果：**
	- 根据数据模型可知：CG_001 来源：`ads_dzcg_bidsum`，CG_003 来源：`ads_dzcg_bidsum`
	- 因此可以直接组装 SQL：
```
SELECT
    SUM(gkcaigouamount) AS public_procurement_amount,
    SUM(caigouAmount) - SUM(gkcaigouamount) AS non_public_procurement_amount,
    ROUND(SUM(gkcaigouamount) / NULLIF(SUM(caigouAmount), 0) * 100, 2) AS public_ratio,
    ROUND((SUM(caigouAmount) - SUM(gkcaigouamount)) / NULLIF(SUM(caigouAmount), 0) * 100, 2) AS non_public_ratio
FROM ads_dzcg_bidsum
WHERE year = #{year};
```
- **SQL执行结果：**
	- 调用SQL执行工具，对ADS表执行查询，得到结果：公开采购金额、非公开采购金额、公开采购占比、非公开采购占比。
- **图表可视化与答案生成：**
	- 利用饼图展示金额构成占比；
	- LLM根据结果自动生成描述，例如：2025 年采购管理域中，公开采购金额为 X 亿元，占总采购金额的 Y%；非公开采购金额为 Z 亿元，占比为 W%。整体上，公开采购仍为主要构成部分，但非公开采购仍占有一定比例，建议结合管理目标进一步评估公开采购比例是否达标。

### 智能体执行过程
- **意图识别执行结果，输出结构化意图：**
	- Domain：采购管理域
	- Scenario：采购结构分析
	- Object：采购金额结构
	- Goal：分析公开与非公开采购金额构成
	- IntentType：看构成
- **上下文获取与指标匹配执行结果：**
	- 根据业务模型识别到该问题对应的业务活动是：采购执行结果统计、采购方式结构分析；
	- 根据指标模型匹配：`CG_001 电子采购中标金额`、`CG_003 公开采购中标金额`
	- 推导：非公开采购金额 = `CG_001 - CG_003`
	- 因此，业务被转化为用指标描述的业务：对电子采购总金额和公开采购金额进行计算，并进一步推导非公开采购金额，分析两者的金额构成与占比。
- **SQL转换执行结果：**
	- 根据数据模型可知：CG_001 来源：`ads_dzcg_bidsum`，CG_003 来源：`ads_dzcg_bidsum`
	- 因此可以直接组装 SQL：
```
SELECT
    SUM(gkcaigouamount) AS public_procurement_amount,
    SUM(caigouAmount) - SUM(gkcaigouamount) AS non_public_procurement_amount,
    ROUND(SUM(gkcaigouamount) / NULLIF(SUM(caigouAmount), 0) * 100, 2) AS public_ratio,
    ROUND((SUM(caigouAmount) - SUM(gkcaigouamount)) / NULLIF(SUM(caigouAmount), 0) * 100, 2) AS non_public_ratio
FROM ads_dzcg_bidsum
WHERE year = #{year};
```
- **SQL执行结果：**
	- 调用SQL执行工具，对ADS表执行查询，得到结果：公开采购金额、非公开采购金额、公开采购占比、非公开采购占比。
- **图表可视化与答案生成：**
	- 利用饼图展示金额构成占比；
	- LLM根据结果自动生成描述，例如：2025 年采购管理域中，公开采购金额为 X 亿元，占总采购金额的 Y%；非公开采购金额为 Z 亿元，占比为 W%。整体上，公开采购仍为主要构成部分，但非公开采购仍占有一定比例，建议结合管理目标进一步评估公开采购比例是否达标。
